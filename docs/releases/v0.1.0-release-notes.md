# OrbitFabric Studio v0.1.0 - Release Notes

Release candidate notes for `v0.1.0 - Read-only Mission Project Viewer`.

---

## Purpose

This release establishes the first runnable OrbitFabric Studio application.

The goal is deliberately narrow:

```text
Open -> Inspect
```

Studio v0.1.0 can open a local OrbitFabric workspace, inspect its structural layout, display source/scenario files read-only and run fixed OrbitFabric Core status commands.

It does not become a second model engine.

OrbitFabric Core remains authoritative.

---

## Added

- Tauri 2 desktop application shell.
- React and TypeScript frontend.
- Vite build setup.
- Monaco Editor integration in read-only mode.
- Local workspace directory opening.
- Structural OrbitFabric workspace inspection.
- Expected Mission Model file detection.
- Scenario YAML file detection.
- Generated documentation, report, log and runtime artifact directory detection.
- Source model, scenario source, derived report and generated output labels.
- Bounded read-only text file loading.
- Workspace containment guard for file viewing.
- Controlled OrbitFabric Core command status panel.
- Fixed `orbitfabric --version` command.
- Fixed `orbitfabric inspect mission <mission_dir>` command.
- Raw stdout, stderr, exit-code and success status display.
- Technical temporary application icon and generated Tauri icon set.
- Lockfiles for reproducible local builds.

## Verified Smoke Tests

Manual smoke testing covered:

```text
Open workspace -> examples/demo-3u
Detect mission directory
Detect scenario sources
Open source model files read-only

```


---

## Explicit Non-goals

v0.1.0 does not include:

- editing;
- visual model editing;
- graph view;
- scenario execution;
- generator workflows;
- plugin support;
- independent validation;
- lint diagnostics UI;
- deep semantic parsing inside Studio;
- mission-control UI;
- live telemetry;
- command uplink;
- ground segment behavior;
- arbitrary command execution;
- arbitrary OrbitFabric CLI argument entry;
- external compatibility claims.

---

## Known Limitations

- The application icon is a temporary technical asset.
- The Core executable path is manually configurable and is not persisted yet.
- Core command output is displayed as raw process output only.
- Generated artifact discovery is structural and best-effort.
- Studio does not determine artifact freshness.
- Studio does not parse Mission Model semantics.
- Studio does not surface lint findings as structured diagnostics yet.
- Studio does not provide installer packaging yet.
