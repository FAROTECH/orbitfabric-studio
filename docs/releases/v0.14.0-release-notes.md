# OrbitFabric Studio v0.14.0 Release Notes

Release: `v0.14.0 - Artifact Traceability Integration`

Status: release candidate.

## Summary

v0.14.0 closes the Artifact Traceability Integration milestone for OrbitFabric Studio.

The milestone turns the Mission Data Flow Workbench into a stronger read-only traceability surface by connecting Core-reported relationships, validation evidence, scenario data-flow evidence, coverage evidence and generated artifact inventory records into a single Workbench posture.

This is not a north-star visual parity release.

The v0.14.0 north-star gap assessment explicitly records that Studio is moving in the correct direction, but still needs separate future milestones for Mission Cockpit realization and read-only graph Workbench maturity.

## Added

- Added v0.14.0 Artifact Traceability Integration planning and boundary documentation.
- Added v0.14.0 release checklist.
- Added a read-only Mission Data Flow Workbench traceability model foundation.
- Added traceability groups, traceability links, traceability endpoints and summary counts.
- Added traceability link generation for Core-reported relationship manifest records.
- Added traceability link generation for Core simulation data-flow evidence records.
- Added traceability link generation for generated artifact inventory entries.
- Added traceability link generation for Core lint or dashboard validation evidence.
- Added traceability link generation for Core coverage summary evidence.
- Added compact Workbench Inspector traceability blocks for selected Core-reported nodes, canvas edges and evidence records.
- Added a generated artifact inventory bridge for read-only linkage between Generated Artifacts and the Workbench.
- Added Workbench snapshot linkage for generated artifact records and generated output traceability links.
- Added a route-level Workbench traceability posture panel.
- Added north-star gap assessment and roadmap recalibration after the v0.14.0 traceability work.

## Changed

- Updated the Mission Data Flow Workbench header, toolbar and evidence summary to expose traceability counts without adding graph semantics.
- Updated the Generated Artifact Explorer to publish read-only inventory evidence to the Workbench.
- Refined the Workbench route copy and visual hierarchy toward the Mission Data Flow Workbench north-star direction.
- Recalibrated the roadmap so Mission Cockpit realization and read-only graph Workbench maturity precede plugin-aware Studio surfaces.
- Kept React Flow and graph-library adoption deferred.

## Explicit boundaries

v0.14.0 does not introduce:

- React Flow.
- Graph library adoption.
- Graph editing.
- Drag/drop graph persistence.
- Layout-as-mission-semantics.
- Mission Model authoring.
- YAML editor behavior.
- Generated artifact mutation.
- Artifact regeneration.
- Command uplink.
- Live telemetry.
- Operational ground behavior.
- Private YAML semantic parsing.
- Private scenario YAML interpretation.
- Private log-derived evidence.
- Private relationship inference.
- Private data-flow inference.
- Private coverage calculation.
- Private mission health calculation.
- Private readiness calculation.
- Private model completeness calculation.
- Autonomy implementation.
- Plugin execution.
- Plugin marketplace.
- Arbitrary command execution.

## North-star position

v0.14.0 is an architectural traceability milestone.

It does not claim parity with either accepted north-star reference.

The next roadmap direction is:

```text
v0.15.0 - Mission Cockpit Realization
v0.16.0 - Read-only Graph Workbench Engine
v0.17.0 - Workbench Evidence Console
v0.18.0 - Authoring Readiness Assessment
v0.19.0 or later - Plugin-aware Studio Surface
```

## Validation

Required local validation before merge:

```bash
npm run build
cargo check --manifest-path src-tauri/Cargo.toml
```

Recommended smoke validation:

```bash
npm run tauri:dev
```

Manual smoke path:

- Open a valid OrbitFabric workspace.
- Confirm Mission Cockpit renders.
- Confirm Mission Data Flow Workbench renders.
- Confirm Generated Artifacts renders.
- Inspect generated artifacts.
- Confirm generated artifact evidence appears in the Workbench after inspection.
- Select Workbench records and confirm Inspector traceability blocks remain read-only.
- Confirm unavailable traceability is explicit.
- Confirm no private relationship or data-flow inference is shown.
