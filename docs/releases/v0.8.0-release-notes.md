# OrbitFabric Studio v0.8.0 - Ground Integration Artifact Viewer

## Status

```text
Released baseline: v0.8.0 - Ground Integration Artifact Viewer
Previous baseline: v0.7.2 - Core-derived Dashboard UX Realization
Nature: read-only ground-facing generated artifact inspection slice
Primary loop: Inspect Ground Artifacts -> Trace Provenance -> Understand Intended Integration Use
```

## Purpose

v0.8.0 introduces the first dedicated Ground Integration Artifact Viewer surface in OrbitFabric Studio.

The surface is intentionally read-only and non-operational.

Its purpose is to make generated ground-facing artifacts easier to locate, classify, preview and explain without turning Studio into a ground segment, mission control interface, telemetry archive, live decoder or command uplink surface.

Studio remains downstream from OrbitFabric Core.

OrbitFabric Core remains authoritative for generated artifacts and contract meaning.

## Implemented scope

- Added a dedicated Ground surface in the Studio navigation.
- Converted Ground from a reserved future placeholder into an implemented read-only surface.
- Added a Ground Integration Artifact Viewer component.
- Reused the existing generated artifact inventory backend command.
- Filtered generated artifacts to `artifact_class === "ground"`.
- Grouped ground-facing artifacts by conservative family:
  - ground contract manifest;
  - JSON dictionaries;
  - CSV dictionaries;
  - ground documentation;
  - unknown ground-facing artifacts.
- Preserved visibility for unknown ground-facing artifacts.
- Added known/unknown status display for ground artifacts.
- Added preview eligibility display for ground artifacts.
- Added read-only preview for previewable generated ground artifacts.
- Reused the existing `read_text_file` preview path.
- Bound selected ground artifacts to the existing Inspector panel.
- Preserved explicit non-operational boundary copy.

## Validation boundary

The viewer identifies generated ground-facing artifacts by conservative artifact class, path and extension rules.

It does not interpret dictionary contents semantically.

It does not claim compatibility with any external ground system.

It does not validate operational readiness.

It does not claim live decoder availability.

It does not claim command uplink capability.
