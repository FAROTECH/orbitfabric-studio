# OrbitFabric Studio v0.2.0 Release Notes

Release: `v0.2.0 - Validation and Diagnostics Workbench`

Status: release readiness candidate

OrbitFabric Studio v0.2.0 extends the v0.1.0 read-only workspace viewer into a Core-derived validation and diagnostics workbench.

The implemented product loop is:

```text
Open -> Inspect -> Validate -> Understand
```

Studio remains downstream from OrbitFabric Core.

The Mission Model remains the source of truth.

OrbitFabric Core remains authoritative for validation, linting and engineering semantics.

---

## Core baseline

The validated Core baseline for this release is:

```text
OrbitFabric Core 0.8.0
```

Studio v0.2.0 consumes the fixed Core lint command:

```text
orbitfabric lint <mission_dir> --json <report_path>
```

The JSON lint report is treated as a derived report produced by Core.

Studio does not parse stdout as diagnostics when the JSON report exists.

Raw stdout, stderr and exit code remain visible for transparency.

## Added

v0.2.0 adds:

```text
fixed Core lint command invocation
Core JSON lint report path display
Core JSON lint report availability display
typed Core lint report model
Core validation summary panel
Core-derived result display
mission and Mission Model version display
Core version display
errors, warnings and info counts
read-only Core findings list
safe source-file links from findings
read-only opening of linked source files
layout handling for long entry names
```

The findings list displays only fields provided by the Core JSON report:

```text
severity
code
file
domain
object_id
message
suggestion
```

A finding file reference becomes clickable only when the Core-provided file name exactly matches a known source model file detected in the selected workspace.

## Known limitations

Current Core JSON lint reports do not provide:

```text
line metadata
column metadata
schema version
report timestamp
Git SHA
JSON Schema URL
absolute source path
```

Studio therefore does not display line or column jumps.

Studio does not infer missing file references from domains, object IDs or messages.

Studio links a finding to a file only when the Core-provided `file` field exactly matches a detected source model file.

Ground artifacts introduced by OrbitFabric Core v0.8.0 remain out of scope for Studio v0.2.0.

---

## Release boundary

This release makes Core validation results inspectable.

It does not make Studio a validator.

It does not make Studio a model editor.

It does not make Studio a ground segment.
