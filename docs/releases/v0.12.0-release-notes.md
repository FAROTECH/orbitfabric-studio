# OrbitFabric Studio v0.12.0 - Mission Data Flow Workbench Foundation

## Status

```text
Release: v0.12.0
Milestone: Mission Data Flow Workbench Foundation
Status: release candidate
Starting baseline: v0.11.1 - Runtime UI triage hotfix
```

v0.12.0 introduces the first controlled Mission Data Flow Workbench foundation in OrbitFabric Studio.

The milestone moves Studio closer to the Reference B UI north star while preserving the mandatory Core-derived boundary.

Studio remains downstream from OrbitFabric Core.

Studio consumes Core-derived reports and generated artifact state. It does not create mission semantics, infer private data-flow relationships or implement editable graph behavior.

## Added

- Added a dedicated Mission Data Flow Workbench data model foundation.
- Added explicit Workbench source summaries, lanes, records, counts and boundary metadata.
- Added an isolated read-only Mission Data Flow Workbench surface shell.
- Embedded the Workbench foundation inside the existing Mission Cockpit during the early implementation slice.
- Aligned the Workbench shell layout with Reference B from the Studio UI North Star document.
- Added a Workbench tab strip with Graph View, YAML View, Scenario Runner and Data Flow Evidence entries.
- Added a central read-only canvas-like Graph View foundation.
- Added right-side Workbench Inspector placeholder content.
- Added lower Workbench evidence panels for Scenario Timeline and Lint / Validation Results.
- Added Core-derived canvas node and edge rendering from reported relationship manifest records.
- Added a dedicated Mission Data Flow Workbench route frame.
- Wired the dedicated Mission Data Flow Workbench surface into the Studio shell.
- Added a dedicated sidebar entry for Data Flow Workbench.
- Added a shell command bar action for Inspect Data Flow.
- Expanded Workbench input sources to include Core model summary, Core entity index, Core dashboard summary, Core coverage summary and Core simulation report data-flow evidence.
- Added relationship manifest consumption for Workbench relationship lanes when Core reports `relationship_manifest.json` content.
- Added App-level relationship manifest persistence for the dedicated Workbench surface.
- Added shell status bar support for the dedicated Workbench surface.
- Added v0.12.0 planning, source expansion, persistence, validation and dedicated surface documentation.

## Changed

- The Mission Cockpit now exposes a read-only Workbench foundation and the shell also exposes the dedicated Workbench surface.
- The Workbench reuses existing Studio badge, provenance, navigation and status vocabulary.
- Relationship records shown in the Workbench are limited to Core-reported relationship manifest records.
- Scenario data-flow evidence shown in the Workbench is limited to Core simulation report `data_flow_evidence` records.
- Coverage records shown in the Workbench are limited to Core coverage summary records.
- The current Studio implementation baseline moves from `v0.11.1` to `v0.12.0`.

## Validation

Required local validation before release closure:

```bash
npm run build
cargo check --manifest-path src-tauri/Cargo.toml
```

Recommended local metadata validation:

```bash
npm install --package-lock-only
cargo check --manifest-path src-tauri/Cargo.toml
```

If generated lockfiles change, commit the lockfile updates before merging the release closure PR.

Recommended manual smoke path:

1. Open a valid OrbitFabric workspace.
2. Confirm the Mission Cockpit renders.
3. Confirm the Data Flow Workbench entry appears in the shell navigation.
4. Confirm the Inspect Data Flow command bar action opens the dedicated Workbench surface.
5. Run or load Core model summary.
6. Run or load Core entity index.
7. Run or load Core relationship manifest.
8. Run or load Core dashboard summary.
9. Run or load Core coverage summary.
10. Run or load a Core simulation report with data-flow evidence, if available.
11. Confirm the dedicated Mission Data Flow Workbench remains visible and read-only.
12. Confirm relationship records remain visible after switching the current Core output to another supported report.
13. Confirm relationship records reset after changing workspace.

## Not Included

v0.12.0 does not introduce:

- React Flow;
- graph library adoption;
- graph editing;
- drag/drop graph behavior;
- layout engine semantics;
- Mission Model authoring;
- generated artifact mutation;
- plugin behavior;
- live telemetry;
- command uplink;
- operational ground behavior;
- private YAML semantic parsing;
- private relationship inference;
- private data-flow inference;
- mission health calculation;
- readiness calculation;
- completeness calculation;
- Autonomy implementation.

## Boundary

Correct pattern preserved:

```text
OrbitFabric Core emits a structured output.
Studio consumes and renders it.
```

Incorrect pattern still forbidden:

```text
Studio reimplements Core semantics because the required output is missing.
```

If Core does not report a value, Studio must continue to render it as unavailable, not reported, reserved, diagnostic or omit it.
