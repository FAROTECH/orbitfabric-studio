# OrbitFabric Studio v0.5.0 Release Notes

Release: `v0.5.0 - Generated Artifact Explorer`

## Summary

OrbitFabric Studio v0.5.0 adds the first Generated Artifact Explorer.

The implemented product loop is now:

```text
Open -> Inspect -> Validate -> Navigate -> Explain Relationships -> Inspect Generated Artifacts
```

Studio can inspect generated artifacts already present under the selected workspace `generated/` directory, classify them conservatively, group them by broad artifact family and preview supported text artifacts in read-only mode.

The release keeps OrbitFabric Core authoritative for validation, generation and engineering meaning.

## Added

- Generated artifact inventory model.
- Controlled backend command for generated artifact inspection.
- Recursive bounded inspection of `generated/` within the selected workspace.
- Generated artifact classes:
  - reports
  - logs
  - docs
  - runtime
  - ground
  - unknown
- Conservative known artifact classification for Core-documented outputs.
- Unknown artifact visibility without invented semantics.
- Artifact metadata display:
  - file name
  - path
  - relative path
  - extension
  - size
  - artifact class
  - known or unknown status
  - previewable or not-previewable status
- Generated Artifact Explorer panel.
- Read-only preview panel for supported generated text artifacts.
- CSV support for ground dictionary previews.
- Updated Studio landing copy for the v0.5.0 product loop.

## Supported Preview Text Types

The read-only preview path supports bounded text preview for:

```text
yaml
yml
json
md
txt
log
csv
hpp
cpp
h
c
CMakeLists.txt
```

Oversized or unsupported files remain listed but are not previewed.

## Known Limitations

- Studio only inspects artifacts already present under `generated/`.
- Studio does not generate docs, runtime bindings, ground dictionaries, reports or scenario logs.
- Studio does not interpret generated artifact content as Mission Model semantics.
- Studio does not infer runtime behavior from runtime-facing generated artifacts.
- Studio does not infer ground behavior from ground-facing generated artifacts.
- The UI remains vertically long and will need a dedicated navigation/layout refactor after v0.5.0.
