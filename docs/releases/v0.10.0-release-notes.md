# OrbitFabric Studio v0.10.0 Release Notes

## Release

```text
Version: v0.10.0
Name: Mission Cockpit Consolidation
Previous baseline: v0.9.0 - Semantic Navigation & Unified Shell
Next baseline: v0.11.0 - Domain Surfaces & Entity Detail System
Nature: Mission Cockpit visual hierarchy and provenance consolidation milestone
```

## Summary

v0.10.0 consolidates the Mission Dashboard into a clearer Mission Cockpit.

The milestone improves the central cockpit layout, visual hierarchy, provenance vocabulary, reported evidence presentation and shell stability while preserving the Core-derived boundary.

Studio remains downstream from OrbitFabric Core.

OrbitFabric Core remains authoritative for validation, scenario evidence, generated artifacts, contract introspection, entity indexing, relationship semantics and future plugin semantics.

## Added

- Added a dedicated Mission Cockpit implementation path around the mission dashboard.
- Added extracted Mission Cockpit helper/components for cockpit model data, KPI cards, evidence lanes and panel headers.
- Added a denser Mission Cockpit visual hierarchy aligned with the accepted UI north star direction.
- Added reported evidence lanes for contract, validation, scenario, coverage and artifact state.
- Added clearer cockpit shell placement for sidebar, main surface, Inspector and shell status bar.
- Added active navigation tracking by mission-domain item instead of only by destination surface.

## Changed

- Reworked the Mission Dashboard into a Mission Cockpit entry point.
- Updated stale no-workspace hero version copy away from the old v0.7.2 wording.
- Kept the Mission Cockpit dashboard free from the Inspector so the central cockpit has enough horizontal space.
- Preserved Inspector behavior for detail-oriented surfaces.
- Prevented sidebar hash navigation from scrolling the shell when switching surfaces.
- Stabilized the open-workspace shell layout.
- Consolidated duplicated Mission Cockpit shell CSS after the visual follow-up.
- Updated Studio, package lock, Tauri and Cargo metadata to `0.10.0`.

## Explicit Non-goals

v0.10.0 does not include:

- new OrbitFabric Core semantics;
- Mission Model editing;
- generated artifact editing;
- generated artifact mutation;
- visual authoring;
- graph UI;
- React Flow adoption;
- plugin execution;
- plugin marketplace;
- live telemetry;
- telemetry archive behavior;
- command uplink behavior;
- mission control behavior;
- operational ground behavior;
- private YAML semantic parsing;
- private coverage calculation;
- private mission health calculation;
- private readiness calculation;
- private model completeness calculation.

## Boundary

Correct pattern:

```text
OrbitFabric Core emits a structured output.
Studio consumes and renders it.
```

Incorrect pattern:

```text
Studio reimplements Core semantics because the required output is missing.
```

If Core does not report a value, Studio must show `unavailable`, `not reported`, `reserved` or `diagnostic`.
