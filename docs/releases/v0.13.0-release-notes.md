# OrbitFabric Studio v0.13.0 Release Notes

Release: `v0.13.0 - Evidence-integrated Workbench`

Status: tagged technical milestone. GitHub Release publication is deferred until the documentation, roadmap and communication package are reviewed together.

## Summary

v0.13.0 deepens the dedicated Mission Data Flow Workbench introduced in v0.12.0.

The milestone moves the Workbench closer to the Reference B north-star direction by integrating relationship, scenario, validation and coverage evidence into a more coherent read-only engineering surface.

The final hardening pass also stabilizes the Mission Cockpit, promotes Generated Artifacts into a dedicated read-only surface, keeps Core Commands as a diagnostic surface, improves conservative Contacts & Downlink aggregation, improves conservative Commandability aggregation and keeps Autonomy explicitly reserved.

Studio remains downstream from OrbitFabric Core.

Studio does not invent mission semantics, infer private data-flow links, calculate private coverage, parse scenario YAML as evidence, read logs as evidence, mutate generated artifacts or introduce operational ground behavior.

## What changed

v0.13.0 adds an evidence-integrated Workbench layer around the existing dedicated Workbench surface.

The release introduces:

- explicit Workbench evidence kinds for relationship, scenario, validation, coverage and artifact evidence;
- a validation evidence lane in the Workbench model;
- validation evidence records from `CoreLintReport`;
- fallback validation summary records from `CoreDashboardSummary.validation`;
- coverage evidence records for scenario runs, entity coverage, expectation coverage, relationship coverage and unsupported scopes;
- record-level evidence kind provenance metadata;
- read-only local Workbench selection for Core-reported nodes, edges and evidence records;
- a Workbench Inspector that updates from selected Core-derived items;
- raw Core-reported payload preview in the Workbench Inspector;
- Scenario Timeline rendering of Core-reported `data_flow_evidence` fields;
- grouped Validation, Coverage and Relationship evidence in the Lint / Validation Results panel;
- a compact Workbench evidence posture rail;
- Reference B-oriented Workbench visual polish;
- Workbench lint report wiring through both Mission Cockpit and the dedicated Workbench route;
- release hardening for the Home, Mission Cockpit, Data Flow Workbench and Generated Artifacts surfaces;
- a dedicated `GeneratedArtifactsSurface` instead of routing Generated Artifacts through the legacy workspace surface;
- compact Generated Artifact Explorer class selection and read-only preview behavior;
- conservative multi-domain Contacts & Downlink aggregation;
- conservative multi-domain Commandability aggregation;
- explicit reserved Autonomy copy for the v0.13.0 baseline.

## Reference B alignment

v0.13.0 does not claim that the Workbench is the final north-star UI.

It does move the implementation closer to Reference B through clearer hierarchy between canvas, Inspector, scenario evidence and validation evidence, denser evidence panels, more explicit source and provenance language, a compact evidence posture rail and selectable Core-reported nodes and records.

The final QA pass confirmed that additional layout refinement remains possible, especially for the Data Flow Workbench and Generated Artifacts after inspection, but those items are v0.14.0 polish and traceability work, not v0.13.0 blockers.

## Core-derived evidence boundary

v0.13.0 preserves the core rule:

```text
OrbitFabric Core emits a structured output.
Studio consumes and renders it.
```

The Workbench renders only Core-reported or generated artifact inventory data.

Allowed evidence sources include:

- `CoreRelationshipManifest`;
- `CoreSimulationReport`;
- `CoreSimulationReport.data_flow_evidence`;
- `CoreLintReport`;
- `CoreDashboardSummary.validation`;
- `CoreCoverageSummary`;
- generated artifact inventory already exposed by Studio.

If Core does not report a value, Studio renders `not reported`, `unavailable`, `reserved`, `diagnostic` or omits the relation.

## Not included

v0.13.0 does not introduce React Flow, graph library adoption, graph editing, drag/drop graph behavior, layout engine semantics, Mission Model authoring, YAML editor behavior, generated artifact mutation, plugin behavior, plugin execution, live telemetry, telemetry archive behavior, command uplink, mission control behavior, operational ground behavior, private YAML semantic parsing, private scenario YAML interpretation, private log-derived evidence, private relationship inference, private data-flow inference, private coverage calculation, private mission health calculation, private readiness calculation, private model completeness calculation, command authorization, command scheduling, command execution or Autonomy implementation.
