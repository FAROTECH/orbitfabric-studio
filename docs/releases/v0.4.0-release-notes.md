# OrbitFabric Studio v0.4.0 Release Notes

Release: `v0.4.0 - Relationship Surface`

## Summary

OrbitFabric Studio v0.4.0 adds the first Core-derived Relationship Surface.

The implemented product loop is now:

```text
Open -> Inspect -> Validate -> Navigate -> Explain Relationships
```

Studio can now run the fixed OrbitFabric Core relationship manifest export command, load the Core-owned `relationship_manifest.json` report, inspect relationship types, inspect relationship records and explain a selected relationship record without creating private relationship semantics.

## Core Surface Consumed

v0.4.0 consumes the OrbitFabric Core v1.0.0 Relationship Manifest Surface:

```text
orbitfabric export relationship-manifest <mission_dir> --json <studio_report_path>
```

Expected report identity:

```text
kind: orbitfabric.relationship_manifest
manifest_version: 0.1-candidate
status: candidate
```

## Added

- Fixed backend command for Core `export relationship-manifest`.
- Studio-controlled relationship manifest report path.
- TypeScript model and parser for Core `relationship_manifest.json`.
- `Run export relationship-manifest` UI action.
- Relationship Manifest summary panel.
- Boundary labels for the Relationship Manifest Surface.
- Raw `relationship_manifest.json` preview.
- Relationship type summary and filters.
- Relationship record navigation and filters.
- Selected relationship explanation panel.
- Explicit provenance and boundary statements for selected relationship records.

relationship records invented by Studio

## Known Limitations

- Endpoint linking from relationship records to entity index records is not implemented in v0.4.0.
- Source line and column navigation are not implemented because Core v1.0.0 does not expose source locations for relationship records.
- Graph rendering is intentionally not included.
- Relationship records are displayed read-only exactly as reported by Core.
