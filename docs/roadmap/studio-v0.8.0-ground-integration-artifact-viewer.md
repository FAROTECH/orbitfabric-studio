# OrbitFabric Studio v0.8.0 - Ground Integration Artifact Viewer

## Status

```text
Milestone: v0.8.0 - Ground Integration Artifact Viewer
Status: Planned
Nature: read-only ground-facing generated artifact inspection slice
Primary loop: Inspect Ground Artifacts -> Trace Provenance -> Understand Intended Integration Use
Starting baseline: v0.7.2 - Core-derived Dashboard UX Realization
```

## Purpose

v0.8.0 introduces a dedicated read-only surface for inspecting ground-facing artifacts generated by OrbitFabric Core.

The purpose is to make generated ground integration outputs easier to find, classify, preview and explain without turning Studio into a ground segment.

Studio remains downstream from OrbitFabric Core.

The Mission Model remains the source of truth.

Ground-facing generated artifacts remain disposable outputs derived from the contract.

## Starting baseline

v0.7.2 provides the required Studio foundation:

- compact cockpit shell;
- primary surface navigation;
- contextual inspector pattern;
- read-only generated artifact inventory;
- conservative generated artifact classification;
- preview support for supported text artifacts;
- `ground` artifact class in the generated artifact model;
- reserved Ground surface placeholder;
- clear source, Core-derived, generated and UI-state distinction.

v0.8.0 must build on this foundation without replacing it.

## Core alignment rule

Correct pattern:

```text
OrbitFabric Core generates ground-facing artifacts.
Studio inventories, classifies, previews and explains them read-only.
```

Incorrect pattern:

```text
Studio interprets ground semantics privately or behaves like a ground system.
```

Studio must not invent compatibility, operational readiness, live decoding behavior, telemetry archive semantics, command uplink behavior or mission-control behavior.

## Allowed data sources

The Ground surface may consume only:

```text
WorkspaceInspection
generated artifact inventory
supported generated text artifact preview
known Core-documented generated ground paths
local UI selection state
explicit unavailable state
explicit reserved/future state
```

The Ground surface must not read Mission Model YAML semantically.

The Ground surface must not derive ground behavior from artifact contents unless Core later emits a stable machine-readable manifest defining that meaning.

## Initial generated ground artifact families

The first implementation should recognize and present these generated ground-facing families when present under `generated/ground/`:

```text
ground contract manifest
ground README / generated documentation
JSON dictionaries
CSV dictionaries
Markdown ground dictionary documentation
unknown ground-facing generated files
```

Initial known paths already recognized by the generated artifact layer include:

```text
ground/generic/ground_contract_manifest.json
ground/generic/README.md

ground/generic/dictionaries/command_dictionary.json
ground/generic/dictionaries/event_dictionary.json
ground/generic/dictionaries/fault_dictionary.json
ground/generic/dictionaries/data_product_dictionary.json
ground/generic/dictionaries/packet_dictionary.json
ground/generic/csv/telemetry_dictionary.csv
ground/generic/csv/command_dictionary.csv
ground/generic/csv/event_dictionary.csv
ground/generic/csv/fault_dictionary.csv
ground/generic/csv/data_product_dictionary.csv
ground/generic/csv/packet_dictionary.csv
ground/generic/ground_dictionaries.md
```

These paths are identity hints only.

They do not authorize Studio to claim operational compatibility, ground runtime behavior or live decoding capability.

## Target surface behavior

The v0.8.0 Ground surface should provide:

- a dedicated Ground surface reachable from the existing sidebar;
- removal of the purely reserved Ground state once the surface is implemented;
- clear heading identifying the surface as `Ground Integration Artifact Viewer`;
- generated artifact inventory refresh action or reuse of existing generated artifact inspection flow;
- filtered view of `artifact_class === ground` artifacts;
- family grouping for manifest, dictionaries, CSV, documentation and unknown ground files;
- known/unknown status for each artifact;
- preview eligibility for each artifact;
- read-only preview for supported text artifacts;
- empty state when no generated ground artifacts are present;
- warning state when generated artifacts exist but no ground-facing artifacts are found;
- contextual inspector binding for selected ground artifacts;
- explicit boundary copy stating that the surface is not a ground segment.

## Suggested UI taxonomy

A selected ground artifact should expose:

```text
name
relative path
artifact class
family
known or unknown status
preview status
size
extension
provenance source
provenance detail
```

Possible families:

```text
manifest
dictionary-json
dictionary-csv
documentation
unknown-ground-artifact
```

Family classification may be based only on conservative path and extension rules.

## Preview rules

Preview is allowed only for supported text artifacts already accepted by the backend read-only viewer.

Preview must remain read-only.

Preview must not imply:

- validation;
- source-of-truth status;
- operational readiness;
- ground system compatibility;
- decoder availability;
- command uplink capability;
- live telemetry behavior.

## Empty and unavailable states

The Ground surface must handle these states explicitly:

```text
no workspace open
workspace has no generated directory
generated directory exists but contains no ground artifacts
ground artifacts exist but none are previewable
artifact preview failed
artifact is unknown but still visible
```

Unknown ground-facing files must remain visible.

Unknown does not mean invalid.

Unknown means Studio does not have a documented Core path or conservative identity rule for the file.

## Implementation constraints

`src/App.tsx` remains sensitive.

Any v0.8.0 implementation PR must keep changes local, small and reviewable.

No PR may replace `src/App.tsx` wholesale.

No PR may remove existing implemented surfaces.

No PR may duplicate generated artifact inventory behavior in a divergent model.

The Ground surface should reuse the existing generated artifact inventory model where possible.

CSS should remain scoped to Ground/workbench-specific classes where possible.

The repository uses `src/styles.css`, not `src/App.css`.

## Explicit non-goals

v0.8.0 must not implement:

- ground segment behavior;
- mission control behavior;
- command uplink behavior;
- live telemetry behavior;
- telemetry archive behavior;
- live decoder behavior;
- Yamcs replacement;
- OpenC3 replacement;
- compatibility claim with any external ground system unless implemented, tested and documented;
- generated artifact editing;
- generated artifact mutation;
- Mission Model editing;
- source file mutation;
- private ground semantics;
- YAML semantic parsing;
- dictionary semantic interpretation beyond conservative display;
- operational database behavior;
- runtime execution;
- arbitrary command execution;
- arbitrary OrbitFabric CLI argument entry;
- graph UI;
- React Flow adoption;
- plugin execution;
- authoring.

## Candidate implementation PR sequence

### PR 1 - Roadmap and boundary note

Documentation-only PR.

Deliverables:

- this roadmap note;
- no code changes;
- no version bump;
- no CHANGELOG release entry.

### PR 2 - Ground surface shell

Introduce a dedicated Ground surface using existing navigation and reserved surface structure.

Deliverables:

- active Ground surface state;
- Ground surface header;
- empty/unavailable states;
- no artifact-specific parsing beyond existing inventory availability.

### PR 3 - Ground artifact filtering and grouping

Use generated artifact inventory to show only ground-facing artifacts.

Deliverables:

- ground artifact filtering;
- family grouping;
- known/unknown display;
- preview status display;
- no semantic dictionary interpretation.

### PR 4 - Ground preview and inspector binding

Reuse existing read-only preview behavior and bind selected ground artifacts to the Inspector.

Deliverables:

- previewable ground artifacts open read-only;
- selected artifact details appear in Inspector;
- unsupported or oversized artifacts remain listed but not previewed.

### PR 5 - UX polish and v0.8.0 closure

Finalize cockpit styling and release documentation.

Deliverables:

- compact Ground cockpit surface;
- release notes;
- changelog entry;
- checklist;
- metadata bump only at release closure.

## Exit criteria

v0.8.0 is complete when Studio can inspect generated ground-facing artifacts through a dedicated read-only surface, preview supported dictionaries and manifests, expose provenance and intended integration use, and avoid any ground segment or operational claim.

The implementation is not complete if it only links to the generic Generated Artifact Explorer without a Ground-specific inspection surface.

The implementation is invalid if it claims live ground behavior, command uplink capability, telemetry archive behavior, live decoding behavior or compatibility with external ground systems without explicit implementation and validation.
