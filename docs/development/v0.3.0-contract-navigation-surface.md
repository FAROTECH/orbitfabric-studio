# OrbitFabric Studio v0.3.0 - Contract Navigation Surface

This document records the implemented development slices for `v0.3.0 - Contract Navigation Surface`.

The purpose of v0.3.0 is to make OrbitFabric Mission Data Contract domains and entities navigable in Studio by consuming Core-owned derived reports.

Primary loop:

```text
Open -> Inspect -> Validate -> Navigate
```

v0.3.0 builds on v0.2.0.

It does not change the source of truth.

The Mission Model remains the authoritative source.

OrbitFabric Core remains authoritative for model loading, validation, linting, introspection and entity indexing.

Studio remains downstream.

---

## Baseline

v0.2.0 implemented:

```text
local workspace opening
structural workspace inspection
read-only file viewer
fixed Core --version command
fixed Core inspect mission command
fixed Core lint mission command
Core JSON lint report preview
Core validation summary
Core findings list
safe source-file links from Core findings
raw stdout / stderr / exit-code visibility
```

v0.3.0 extends that baseline with contract navigation.

It does not introduce editing, private YAML parsing, private validation rules, relationship inference or graph rendering.

---

## Core Surfaces Consumed

v0.3.0 consumes two Core-owned derived report surfaces when available.

### Core v0.8.1 Contract Introspection Surface

Domain navigation is based on:

```bash
orbitfabric export model-summary <mission_dir> --json <studio_controlled_report_path>
```

Expected derived report:

```text
model_summary.json
kind: orbitfabric.model_summary
summary_version: 0.1
```

Studio may consume:

```text
mission identity
source mission directory
boundary flags
domain-level counts
source file metadata
required
present
count
count_provenance
```

This surface answers:

```text
What contract domains are present in this mission?
```

### Core v0.8.2 Entity Index Surface

Entity navigation is based on:

```bash
orbitfabric export entity-index <mission_dir> --json <studio_controlled_report_path>
```

Expected derived report:

```text
entity_index.json
kind: orbitfabric.entity_index
index_version: 0.1
```

Studio may consume:

```text
mission identity
source mission directory
boundary flags
domain summaries
total entity count
per-domain entity counts
per-domain model counts
source file metadata
required
present
indexed
model_count
entity_count
entity records
```

Each entity record is Core-owned and may expose:

```text
id
domain
entity_type
display_name
source_file
provenance
required_domain
present
```

This surface answers:

```text
What contract entities are defined in this mission?
```

Core v0.8.2 entity-indexed domains are expected to include:

```text
spacecraft
subsystems

telemetry
commands
events
faults
packets
payloads
data_products
contact_profiles
link_profiles
contact_windows
downlink_flows
command_sources
commandability_rules
autonomous_actions
recovery_intents
```

Core v0.8.2 summarized but non entity-indexed domains are expected to include:

```text
mode_transitions
policies
```

Studio must not create synthetic entity records for summarized-only domains.

Studio may show those domains as summarized but not entity-indexed when Core reports them that way.

## Version Compatibility Targets

Studio v0.3.0 handles:

```text
Core v0.8.0:
  lint JSON available
  no model-summary
  no entity-index
  contract navigation disabled with clear explanation

Core v0.8.1:
  model-summary available
  entity-index unavailable
  domain navigation enabled
  entity navigation disabled with clear explanation

Core v0.8.2:
  model-summary available
  entity-index available
  domain navigation enabled
  entity navigation enabled
```

The compatibility check is based on actual command behavior.

The Core executable installed locally may differ from Core source on GitHub.

Studio displays that distinction through command results and fallback states.
