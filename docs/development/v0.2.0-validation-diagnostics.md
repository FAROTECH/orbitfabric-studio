# OrbitFabric Studio v0.2.0 - Development Slices

This document records the implementation slices completed for `v0.2.0 - Validation and Diagnostics Workbench`.

The purpose of v0.2.0 is to make OrbitFabric Core validation results inspectable in Studio without duplicating Core validation or parsing Mission Model semantics inside Studio.

Primary loop:

```text
Open -> Inspect -> Validate -> Understand
```

---

## Baseline

v0.1.0 implemented:

```text
local workspace opening
structural workspace inspection
read-only file viewer
fixed Core --version command
fixed Core inspect mission command
raw stdout / stderr / exit-code display
```

v0.2.0 builds on that baseline.

It does not change the source of truth.

The Mission Model remains authoritative source.

OrbitFabric Core remains authoritative for validation and linting.

Studio remains downstream.

---

## Core v0.8.0 Surface Consumed

The validation surface consumed by v0.2.0 is:

```bash
orbitfabric lint <mission_dir> --json <path>
```

The JSON report is a derived report produced by OrbitFabric Core.

It is not a source model file.

It is not a Studio-owned model.

The report fields consumed by Studio are:

```text
tool
version
mission
model_version
result
loaded
summary
findings
```

Each finding currently provides:

```text
severity
code
file
domain
object_id
message
suggestion
```

Current limitations:

```text
no line metadata
no column metadata
no schema version
no report timestamp
no Git SHA
no JSON Schema URL
no absolute source path
```

Studio does not invent those fields.

Completed in PR #13:

```text

read-only opening of matched files through the existing viewer

```

A file reference is clickable only when the Core-provided `file` value exactly matches a known source model file in the selected workspace.

No line or column highlight is attempted because Core does not provide line or column metadata.

No quick fixes are provided.

No suppressions are provided.

## Ground Integration Compatibility Note

OrbitFabric Core v0.8.0 introduced:

```text
GroundContract intermediate model
orbitfabric gen ground <mission_dir>
generated/ground/generic/
ground_contract_manifest.json
JSON ground dictionaries
CSV ground dictionaries
generated ground README.md
ground_dictionaries.md
```

Studio v0.2.0 remains focused on validation and diagnostics.

Ground artifact generation and ground artifact inspection are not part of v0.2.0.

A future milestone may inspect generated or ground-facing artifacts, but v0.2.0 does not expose that workflow.
