# OrbitFabric Studio v0.2.0 - Development Slices

This document records the planned implementation slices for `v0.2.0 - Validation and Diagnostics Workbench`.

The purpose of v0.2.0 is to make OrbitFabric Core validation results inspectable in Studio without duplicating Core validation or parsing Mission Model semantics inside Studio.

Primary loop:

```text
Open -> Inspect -> Validate -> Understand
```

---

## Baseline

v0.1.0 implemented:

```text
local workspace opening
structural workspace inspection
read-only file viewer
fixed Core --version command
fixed Core inspect mission command
raw stdout / stderr / exit-code display
```

v0.2.0 builds on that baseline.

It does not change the source of truth.

The Mission Model remains authoritative source.

OrbitFabric Core remains authoritative for validation and linting.

Studio remains downstream.

---

## Core v0.8.0 Surface Consumed

The validation surface consumed by v0.2.0 is:

```bash
orbitfabric lint <mission_dir> --json <path>
```

The JSON report is a derived report produced by OrbitFabric Core.

It is not a source model file.

It is not a Studio-owned model.

The current report fields are:

```text
tool
version
mission
model_version
result
loaded
summary
findings
```

Each finding currently provides:

```text
severity
code
file
domain
object_id
message
suggestion
```

Current limitations:

```text
no line metadata
no column metadata
no schema version
no report timestamp
no Git SHA
no JSON Schema URL
```

Studio must not invent those fields.

## Slice 2 - Controlled Core Lint Command

Planned in PR B:

```text
fixed Rust command for Core lint
allowlisted argument set
Studio-controlled JSON report path
raw stdout / stderr / exit-code return
JSON report availability flag
no arbitrary CLI argument entry
```

Expected command shape:

```text
orbitfabric lint <mission_dir> --json <studio_controlled_report_path>
```

The report path should be controlled by Studio.

The source Mission Model must not be modified.

The workspace should not be polluted with temporary reports unless explicitly designed and documented.

This slice may expose the command in the UI as a basic validation action.

It does not yet need to render advanced diagnostics.

---

## Slice 3 - Core Lint Report Model

Planned in PR C:

```text
Rust and/or TypeScript report model
safe JSON loading
report producer identification
summary counts
findings list shape
invalid or unavailable JSON state
```

The model should be permissive enough to display unknown future fields safely, but it must not infer missing semantics.

Recommended TypeScript shape:

```ts
export interface CoreLintReport {
  tool: string;
  version: string;
  mission: string;
  model_version: string;
  result: string;
  loaded: Record<string, number>;
  summary: {
    errors: number;
    warnings: number;
    info: number;
  };
  findings: CoreLintFinding[];
}

export interface CoreLintFinding {
  severity: string;
  code: string;
  file: string | null;
  domain: string | null;
  object_id: string | null;
  message: string;
  suggestion: string | null;
}
```

## Ground Integration Compatibility Note

OrbitFabric Core v0.8.0 introduced:

```text
GroundContract intermediate model
orbitfabric gen ground <mission_dir>
generated/ground/generic/
ground_contract_manifest.json
JSON ground dictionaries
CSV ground dictionaries
generated ground README.md
ground_dictionaries.md
```

Studio v0.2.0 must remain focused on validation and diagnostics.

Ground artifact generation and ground artifact inspection are not part of v0.2.0.

A future milestone may inspect generated or ground-facing artifacts, but v0.2.0 must not expose that workflow.

## Manual Verification Pattern

Each implementation PR should preserve this local verification pattern:

```bash
git fetch
git checkout <branch>
git pull
npm run build
source "$HOME/.cargo/env"
npm run tauri:dev
```

Manual smoke test should include:

```text
open a valid OrbitFabric mission workspace
confirm structural inspection still works
run the fixed Core command introduced by the PR
confirm stdout, stderr and exit code remain visible
confirm source model files remain read-only
confirm src-tauri/gen is not committed
```
