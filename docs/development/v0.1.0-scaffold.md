# OrbitFabric Studio v0.1.0 — Minimal Application Scaffold

This document records the first implementation scaffold for `v0.1.0 — Read-only Mission Project Viewer`.

The purpose of this scaffold is to establish a runnable desktop application baseline without implementing workspace inspection logic yet.

---

## Implemented in this scaffold

```text
Tauri 2 desktop shell
React frontend
TypeScript configuration
Vite build setup
Minimal static Studio shell screen
Minimal Tauri backend entrypoint
Minimal default Tauri capability
```

---

## Deliberately not implemented yet

```text
local directory opening
filesystem traversal
OrbitFabric mission detection
file viewer
Monaco Editor integration
OrbitFabric Core command invocation
generated artifact discovery
validation diagnostics
scenario execution
graph rendering
editing
```

These are intentionally deferred to later v0.1.0 PRs.

---

## Permission boundary

The initial Tauri capability grants only:

```text
core:default
```

No filesystem, dialog or shell execution permissions are granted in this scaffold PR.

Those permissions must be introduced only when the corresponding v0.1.0 feature is implemented and reviewed.

---

## Architecture boundary

This scaffold must not be interpreted as a Studio data model.

The UI is static and boundary-oriented.

It exists only to verify that the desktop shell, frontend build and application entrypoints are in place before real workspace inspection begins.

OrbitFabric Core remains authoritative.

The Mission Model remains the source of truth.

Studio remains downstream.
